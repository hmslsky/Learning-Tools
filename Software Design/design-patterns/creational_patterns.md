# 创建型模式 (Creational Patterns)

创建型模式关注对象的创建过程，它们将对象的创建与使用分离，使系统更加灵活和可维护。本文件整合了所有创建型设计模式的内容。

## 单例模式 (Singleton)

单例模式确保一个类只有一个实例，并提供一个全局访问点。

### 特点
- 保证一个类只有一个实例
- 提供全局访问点
- 延迟初始化（可选）

### 实现方式
1. 懒汉式（线程不安全）
2. 懒汉式（线程安全，同步方法）
3. 饿汉式（线程安全）
4. 双重检查锁（DCL）
5. 静态内部类
6. 枚举

### 应用场景
- 配置管理器
- 数据库连接池
- 线程池
- 缓存管理器

## 原型模式 (Prototype)

原型模式通过克隆已有对象来创建新对象，而不是通过实例化类。

### 特点
- 通过克隆创建对象
- 避免构造函数的约束
- 提高性能

### 实现方式
1. 浅拷贝
2. 深拷贝

### 应用场景
- 对象创建成本较高
- 需要避免工厂类
- 需要避免与产品类层次结构耦合

## 工厂方法模式 (Factory Method)

工厂方法模式定义一个创建对象的接口，让子类决定实例化哪个类。

### 特点
- 将对象的创建延迟到子类
- 符合开闭原则
- 避免与具体产品类耦合

### 实现方式
1. 简单工厂
2. 工厂方法
3. 抽象工厂

### 应用场景
- 不知道具体产品类型
- 需要扩展产品类型
- 需要统一管理对象创建

## 抽象工厂模式 (Abstract Factory)

抽象工厂模式提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类。

### 特点
- 创建一系列相关对象
- 符合开闭原则
- 产品族扩展困难

### 实现方式
1. 抽象工厂接口
2. 具体工厂类
3. 抽象产品接口
4. 具体产品类

### 应用场景
- 需要创建一系列相关对象
- 系统需要独立于产品的创建、组合和表示
- 系统需要配置多个产品族中的一个

## 建造者模式 (Builder)

建造者模式将一个复杂对象的构建与它的表示分离，使同样的构建过程可以创建不同的表示。

### 特点
- 分步骤创建复杂对象
- 相同的构建过程可以创建不同的表示
- 控制对象的创建过程

### 实现方式
1. 抽象建造者
2. 具体建造者
3. 产品类
4. 指挥者

### 应用场景
- 需要生成的对象有复杂的内部结构
- 需要生成的对象内部属性本身相互依赖
- 需要控制对象的创建过程
