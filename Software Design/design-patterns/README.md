# 软件设计 (Software Design) 学习指南

本模块旨在深入浅出地讲解软件设计中的核心概念、原则以及各种设计模式，帮助学习者掌握如何构建灵活、可扩展、易于维护的软件系统。

## 学习模块

本模块包含以下主要学习单元，侧重于设计模式的分类学习：

1.  **创建型模式 (Creational Patterns - `creational_patterns.md`)**
    * **内容提要：** 学习各种创建型设计模式，它们主要用于解决对象的创建问题，使系统在创建对象时更加灵活和可控。本模块涵盖单例模式、工厂方法模式、抽象工厂模式、建造者模式和原型模式等。
    * **整合自：** 原 `B1_DesignPattern/Design Pattern/` 下所有创建型模式的 `.md` 文件。

2.  **结构型模式 (Structural Patterns - `structural_patterns.md`)**
    * **内容提要：** 学习结构型设计模式，它们关注如何将类和对象组合成更大的结构，以形成新的功能，同时保持灵活性和效率。本模块涵盖适配器模式、桥接模式、组合模式、装饰模式、外观模式、享元模式和代理模式等。
    * **整合自：：** 原 `B1_DesignPattern/Design Pattern/` 下所有结构型模式的 `.md` 文件。

3.  **行为型模式 (Behavioral Patterns - `behavioral_patterns.md`)**
    * **内容提要：** 学习行为型设计模式，它们关注对象之间的职责分配和通信方式，以实现更灵活、可扩展的交互。本模块涵盖责任链模式、命令模式、解释器模式、迭代器模式、中介者模式、备忘录模式、观察者模式、状态模式、策略模式、模板方法模式和访问者模式等。
    * **整合自：** 原 `B1_DesignPattern/Design Pattern/` 下所有行为型模式的 `.md` 文件。

4.  **设计图 (`design_diagrams.md`)**
    * **内容提要：** 学习各种软件设计中常用的图表，如UML类图、时序图、用例图等，以及它们在软件设计和文档中的应用。
    * **整合自：** 原 `B1_DesignPattern/Design Diagram/` 下所有图表文件及 `B2_DesignDiagram/` 的内容。
